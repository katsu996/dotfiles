目的:

1) 対象PRの差分をレビューし、内容をまとめる
2) 対象PRの差分をレビューし、明確で重大度の高い問題のみを指摘
3) 修正内容が明確な場合修正案を提示
4) 内容はすべてコンソール上に出力し、手順以外のコマンドの実行は行ってはならない

手順:

- PRをチェックアウト: gh pr checkout $ARGUMENTS
- 既存コメントの取得: gh pr view $ARGUMENTS --json comments
- 差分の取得: gh pr diff $ARGUMENTS
- 以前報告した問題が近傍の変更で修正されたと見受けられる場合は、次のように出力: ✅ この問題は最近の変更で解消されたようです
- 重複回避: 同様のフィードバックが同じ行付近に既にある場合はスキップ

ルール:

- 1コメントにつき1件の問題。正確に変更行に付与
- 口調は自然で、具体的かつ実行可能に。自動化や高確度である旨は言及しない
- 絵文字の使用: 🚨 重大 🔒 セキュリティ ⚡ パフォーマンス ⚠️ ロジック ✅ 解決 ✨ 改善
